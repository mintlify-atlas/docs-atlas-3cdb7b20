This guide walks you through the core UABEA workflow: opening a file, browsing assets, making an edit, and saving.

## Before you begin

Make sure you have [installed UABEA](/installation) and have a Unity game's asset bundle or serialized file on hand. Asset bundles are commonly found in:

- `<game>/[GameName]_Data/StreamingAssets/`
- `<game>/[GameName]_Data/` (for `.assets` files such as `sharedassets0.assets`)

## Open a file

<Steps>
  <Step title="Launch UABEA">
    Run **UABEAvalonia.exe** (Windows) or **UABEAvalonia** (Linux/macOS). The main window opens with an empty asset list.
  </Step>
  <Step title="Open a file">
    Select **File → Open** from the menu bar, then browse to and select your file. UABEA can open:

    | File type | Description |
    |---|---|
    | Asset bundle | Compressed or uncompressed bundle files (`.bundle`, `.ab`, `.unity3d`, and others) |
    | Serialized file | Raw serialized data files (`.assets`, `level*`, `sharedassets*`) |
    | `.decomp` file | A decompressed bundle previously saved by UABEA |

    You can also drag and drop a file onto the UABEA window to open it.

    <Note>
      When you open a compressed asset bundle, UABEA decompresses it to a `.decomp` file alongside the original. This is the file that gets modified when you save changes — the original bundle is overwritten only when you choose **File → Save** or **File → Save As**.
    </Note>
  </Step>
  <Step title="Select a bundle entry (bundles only)">
    If you opened an asset bundle, UABEA shows a list of the files packed inside it. Select the entry you want to inspect — usually a serialized file — and click **OK** to load its asset list.
  </Step>
</Steps>

## Browse and inspect assets

<Steps>
  <Step title="Review the asset list">
    The main window now shows a table of all assets in the file. Each row displays the asset's:

    - **Name** — asset name as stored in the file
    - **Type** — Unity asset type (e.g., `Texture2D`, `AudioClip`, `MonoBehaviour`)
    - **Path ID** — the unique numeric identifier for this asset within the file
    - **Size** — serialized byte size

    Click a column header to sort by that field.
  </Step>
  <Step title="View asset data">
    Select an asset in the list and click **View Data** (or **Info** on some versions). A new panel opens showing the asset's serialized fields as a structured tree. You can read the raw field values here.
  </Step>
</Steps>

## Edit an asset

UABEA's primary editing workflow uses dump files — human-readable text representations of an asset's data that you edit in any text editor and import back.

<Steps>
  <Step title="Export a dump">
    Select the asset you want to edit, then click **Export Dump**. Choose a location to save the `.txt` dump file.

    The dump file contains each field on its own line with its type, name, and value:

    ```text
    0 string m_Name = "my_texture"
    0 int m_Width = 512
    0 int m_Height = 512
    ```
  </Step>
  <Step title="Edit the dump file">
    Open the `.txt` dump file in a text editor. Modify the values you want to change, then save the file.

    <Warning>
      Do not change field names, types, or indentation. Only change the values after the `=` sign. Structural changes will cause import errors.
    </Warning>
  </Step>
  <Step title="Import the dump">
    Back in UABEA, select the same asset and click **Import Dump**. Browse to your edited `.txt` file and confirm. UABEA validates the structure and applies your changes to the in-memory asset.
  </Step>
</Steps>

## Save your changes

<Steps>
  <Step title="Save the file">
    Select **File → Save** to write your changes back to the original file, or **File → Save As** to write to a new location.

    <Tip>
      Use **File → Save As** when editing game files so you keep a backup of the original. You can then replace the original file manually after verifying the edit works.
    </Tip>
  </Step>
  <Step title="Compress the bundle (optional)">
    If you opened a bundle, you can recompress it after saving. Select **File → Compress** and choose a compression type. This reduces file size but takes longer to process.
  </Step>
</Steps>
