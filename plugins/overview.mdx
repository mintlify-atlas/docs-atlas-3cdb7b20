Plugins extend UABEA with asset-type-specific actions such as exporting textures to image files, importing fonts, or extracting audio. Each plugin is a .NET class library compiled as a `.dll` and placed in a `plugins/` directory next to the UABEA executable.

## How plugins are loaded

At startup, `PluginManager` scans the `plugins/` subdirectory next to the executable and loads every `.dll` it finds:

```csharp
public void LoadPluginsInDirectory(string directory)
{
    Directory.CreateDirectory(directory);
    foreach (string file in Directory.EnumerateFiles(directory, "*.dll"))
    {
        LoadPlugin(file);
    }
}
```

Each DLL is inspected via reflection. If a class that implements the `UABEAPlugin` interface is found, its `Init()` method is called to retrieve the plugin's name and the list of options it provides.

## How plugins appear in the UI

After selecting one or more assets in the InfoWindow, click the **Plugin** button. UABEA queries every loaded plugin option with `SelectionValidForPlugin` to determine which actions apply to the current selection. Only valid actions are shown in the list. Selecting an entry calls the corresponding `ExecutePlugin` method.

<Note>
  The Plugin button is only active when at least one asset is selected. If no plugins are compatible with the selected assets, the list will be empty.
</Note>

## Writing your own plugin

You can add support for any asset type by implementing the `UABEAPlugin` and `UABEAPluginOption` interfaces and placing the compiled DLL in the `plugins/` directory.
